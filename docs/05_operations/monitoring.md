# 監視・アラート設定

## 監視対象

| 項目 | 閾値 | アラート先 |
|---|---|---|
| APIレスポンスタイム (p99) | 2秒超過 | メール |
| APIエラーレート (5xx) | 1%超過 | メール |
| DB接続数 | 上限の80%超過 | メール |
| ディスク使用率 | 80%超過 | メール |
| アプリクラッシュ | 1件でも発生 | Sentry通知 |

## ツール

### AWS CloudWatch
- CPU使用率、メモリ使用率、リクエスト数の基本メトリクスを提供
- ダッシュボード: CloudWatch コンソール → Dashboards
- アラーム設定: 閾値超過時に SNS 経由でメール通知

### Sentry
- モバイルアプリ・バックエンドのエラー追跡
- 設定: 環境変数 `SENTRY_DSN` を本番環境に設定
- アラート条件: 新規エラー発生時・エラー頻度が急増した時

## ヘルスチェックエンドポイント

```
GET /health
```

**Response 200**
```json
{
  "status": "ok",
  "db": "ok",
  "version": "1.0.0"
}
```

ECS タスクと ALB のヘルスチェックはこのエンドポイントを使用すること。

## ログ管理

- アプリログ: 構造化ログ（JSON形式）で出力
- ログ保持期間: 30日
- **個人情報のログ出力禁止**: ユーザーの日記内容・メールアドレスはログに含めない
