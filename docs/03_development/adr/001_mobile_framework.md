# ADR-001: モバイルフレームワークの選定

**日付**: 2026-04-01  
**ステータス**: 承認済み  
**決定者**: プロジェクトオーナー

## 背景

nikoroku はiOS・Android両方のユーザーに提供するモバイルアプリが必要。
開発リソースは個人（1名）であるため、単一コードベースでクロスプラットフォーム対応することが望ましい。

## 検討した選択肢

| 選択肢 | メリット | デメリット |
|---|---|---|
| **Flutter** | 高パフォーマンス、ウィジェット豊富、型安全(Dart) | Dartの学習コスト |
| React Native | JavaScript/TypeScript利用可能、エコシステムが大きい | ブリッジのパフォーマンス問題 |
| ネイティブ（iOS+Android別々） | 最高のUX・パフォーマンス | 開発コスト2倍 |

## 決定

**Flutter を採用する。**

## 理由

1. 単一コードベースでiOS・Android両方に対応できる
2. `sqflite` によるオフラインファーストの実装が容易
3. Dart は静的型付けで個人開発でもバグを早期に検出しやすい
4. Riverpod による状態管理が直感的

## 影響

- チームメンバーはDartの学習が必要
- Flutter SDKのバージョン管理として `fvm` の導入を検討する
